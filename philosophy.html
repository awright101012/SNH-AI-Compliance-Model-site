Authority Must Be Earned

Rules own what must be exact. Models handle bounded ambiguity. Every increase in authority is purchased with evidence.

A design philosophy for AI in compliance casework

Compliance casework tempts two kinds of overconfidence. The first says that every decision can be reduced to a fixed rule. The second says that a sufficiently capable model can absorb the rules and handle the rest.

Both miss the actual shape of the work.

The policy is structured, but the evidence is messy. The model can interpret ambiguity, but it does not own the policy. The final answer must be explainable, but an explanation can still be wrong. A system worthy of authority must therefore do more than produce the right label. It must preserve the boundary between policy, evidence, reasoning, verification, and human judgment.

The philosophy of this repository is simple:

The guide defines the operating policy. Software enforces the exact boundaries. Models handle bounded ambiguity. Verifiers challenge the answer. People own what the evidence cannot settle.

Everything else follows from that division.

1. Separate capability from authority

A model may be capable of reading a difficult case and still lack authority to decide it. Capability is empirical: can the model reach the right answer on evidence it has not seen? Authority is institutional: under what conditions may that answer change real work?

The two should never rise together automatically.

In this system, a model earns capability through held-out evaluation. The surrounding architecture earns trust through source control, deterministic gates, schema enforcement, citation checks, fail-closed behavior, and traces. Operational authority comes later and must be granted to a particular lane for a particular use.

A better benchmark score does not repeal the verifier. A fluent explanation does not create missing evidence. A fine-tuned model does not become the rulebook merely because it can recite it.

2. Treat the source of truth as a named choice

The live Model A source is an operational mirror of the UBS Court Research Procedural Guide. The repository says this plainly, including where that operational posture differs from a separate legal-ceiling interpretation.

That honesty matters. Every compliance system answers a question relative to a source of authority. If the source changes—from a procedural guide to counsel-approved legal rules—the labels, tests, traces, and model claims must change with it.

The rulebook is therefore versioned and hashed. Decisions cite stable handbook pointers. The model is not allowed to blur an operational instruction into an independent claim about the law.

Provenance begins before the model call. It begins by naming which truth the system has been built to reproduce.

3. Make abstention a decision, not an embarrassment

Most AI systems treat abstention as a performance defect. In high-stakes casework, refusal is part of competence.

ManualReview means the system found a real boundary: insufficient identity, unresolved relief, missing disposition, contradictory no-hit evidence, uncertain timing, an unrecognized term, a verifier failure, or another condition the current evidence cannot safely settle.

The point is not to maximize review. A model that sends every difficult case to a person has not learned the job. The point is to make certainty conditional on evidence and to measure both kinds of failure:

  • acting when the system should have stopped; and
  • stopping when the evidence already supported a clear answer.

Good abstention is selective. It contracts as evidence, rules, and models improve. It never disappears merely because the product wants a cleaner automation rate.

4. Price every claim with the evidence it requires

The program has several legitimate proof surfaces:

  • deterministic acceptance tests show that code follows the encoded handbook contract;
  • development evaluation supports iteration;
  • a sealed blind run supports a frozen model claim;
  • independent adjudication reduces circularity between engine labels and model scoring;
  • a production guardrails-on run shows the behavior of the complete controlled system;
  • a standalone Model B run shows what the fine-tuned model learned without runtime help.

Each claim has a price. “The engine matches its acceptance contract” is cheaper than “the model generalizes.” “The model generalizes against engine labels” is cheaper than “the model agrees with independent adjudication.” “The system is safe in replay” is cheaper than “the system is approved to decide live cases.”

The correct response to unpaid evidence is not softer marketing language. It is “not yet.”

That is why this repository keeps the ≥0.95 Model A blind gate visible and keeps the Model B handbook retrain deferred. A roadmap may name the next layer. Only evidence may unlock it.

5. Let each lane teach one thing

The three-lane strategy is an argument against premature convergence.

Model S asks how quickly a scaffolded model can learn a bounded policy shape. It is the place for tactical curriculum work, training mechanics, and rapid diagnosis. Its help must be named because its score depends on that help.

Model B asks the harder question: can a fully fine-tuned model become the adjudicator with minimal runtime support? It must operate without supplied rule text, retrieval, or a citation allow-list. If it succeeds, it demonstrates learned capability. It still does not provide the same hot-patchable provenance as Model A.

Model A asks the production question: can a general reasoner work inside a controllable system whose rules, tools, verifier, and trace remain external and inspectable? Its support is a feature of the architecture, not a secret to hide. But the clean model-under-test score must remain separate from the full harness score.

No lane is allowed to borrow another lane’s proof standard. That discipline turns comparison into learning instead of a contest for the largest number.

6. Prefer systems that can reject their own progress

A useful evaluation system must be able to disappoint its builder.

GEPA can improve prompts and bounded configuration, but it cannot alter the handbook, labels, schema, or blind cases. A candidate with promotion failures cannot be crowned merely because its aggregate metric improved. Model B’s curriculum is built, but training waits because the teacher system has not yet completed its own handbook certification.

These gates slow visible progress. They also prevent a common failure: optimizing the measuring instrument until the project appears successful.

The written rejection, the frozen split, the failed report, and the deferred training run are not administrative debris. They are part of the product’s credibility.

The SNH AI comparison shows the opposite risk. In the reviewed rationale export, roughly one in ten rows carried an internal consistency issue, and most high-confidence contradictions shared a retired rule/template family or its derived seven-year pattern. A system can remain operational for months while stale logic compounds inside plausible explanations if the source, precedence, and final reconciliation are not independently enforced.

This is why the model should not own policy memory by default. The open-book architecture makes the current source explicit and lets old logic die when the rule version changes. The standalone model is permitted as an experiment precisely because it must re-earn that behavior on held-out evidence.

7. Keep sensitive data outside the learning story

Raw UBS XML and historical person or case rows are evidence for local analysis. They are not training material and should not appear in tracked narratives.

The model-facing program uses synthetic, masked, or normalized representations. This is not merely a privacy concession. It forces the system to state which facts actually matter to the decision and exposes when a result depends on an unavailable or unsafe feature.

A model that performs only when given uncontrolled historical text has not demonstrated deployable intelligence. It has demonstrated access.

The data boundary therefore strengthens the scientific claim as well as the privacy posture.

8. Build an autonomy ladder whose gates default closed

The path to greater authority should be staged:

  1. Encode: convert the operating guide into a versioned rule and acceptance contract.
  2. Observe: run deterministic and scaffolded experiments on synthetic or normalized cases.
  3. Assist: let Model A reason inside the harness while people inspect the traces and unresolved cases.
  4. Compare: certify the clean Model A lane, train Model B, and measure both on matched held-out evidence.
  5. Delegate narrowly: grant a specific system a specific operational role only after its own safety and capability evidence supports that role.

The ladder is not a schedule. A later stage has no date-based entitlement to begin. Each stage exists partly to create the evidence the next one needs.

This is earned autonomy in its strict sense: not increasing what AI can generate, but increasing what the operation is willing to let a proven system do.

What restraint buys

This philosophy has costs. Versioned rules take work. Synthetic curricula are slower than copying historical rows. Blind splits limit iteration. Independent adjudication is expensive. A fail-closed verifier creates review traffic. Honest lane labels make some results look less impressive.

The return is a system whose failures are legible.

  • A rule error points to a versioned source and test.
  • A normalization error points to an adapter boundary.
  • A reasoning error appears in a held-out comparison.
  • A citation error is blocked before it becomes authority.
  • An evidence gap becomes ManualReview rather than invented certainty.
  • A model replacement does not require rebuilding the policy and trace contract from scratch.

In regulated work, legibility is not secondary to intelligence. It is what makes intelligence governable.

The durable idea

Models will change. The preferred size, provider, tuning method, and tool protocol will change. The handbook will be revised. Some decisions now handled by a reasoner will become deterministic precedent; some rules now encoded may require new interpretation.

The durable architecture is the separation of powers:

source policy is versioned
evidence is bounded
exact rules are executable
ambiguous cases are reasoned over
claims are independently checked
uncertainty has a safe destination
every permitted decision leaves a trace

The goal is not to build an AI that never says “I don’t know.” It is to build a compliance system that always knows what must be true before “I know” is allowed to matter.
